Label
Shadow Copy

From time to time, Windows creates point-in-time copies of your files. This allows you to retrieve older versions of files that you have accidentally deleted or modified. This service is enabled by default on all versions of Windows, but Microsoft only grants access to these copies in the Ultimate, Business and Enterprise editions. This is where ShadowExplorer comes into play. You can find more information about the Shadow Copy on Microsoft's website.

Label
ShadowExplorer

Since the Volume Shadow Copy Service is included in all Windows editions and enabled by default, why not use it? All it takes is an additional tool like ShadowExplorer that can access the shadow storage and make the point-in-time copies accessible to the user.

Limitations

Shadow Copies and ShadowExplorer are in no way a replacement for regular backups! This is very important and cannot be emphasized enough. A Shadow Copy is stored on the same disk as the original data. In the event of a disk failure, all data will be lost, including the Shadow Copies. ShadowExplorer is a good addition to regular backups, but not a replacement.

Frequently Asked Questions

Here are some common questions and answers to help you understand ShadowExplorer better.

When I start ShadowExplorer it is entirely blank, what am I doing wrong?

There seems to be a problem with System Restore when TrueCrypt is being used. The problem persists only as long as a volume is mounted. Try to dismount the disk and restart ShadowExplorer. Another solution is to mount volumes as removable media (in TrueCrypt: Settings -> Preferences... -> Mount volumes as removable media (check) -> OK).

I'm not using TrueCrypt or I did the above and ShadowExplorer is still blank, what can I do now?

Make sure System Restore of Windows Vista/7 is fully working. If you think it is and ShadowExplorer should work as well, please use the contact form. Please include as many details as possible (Windows Version, Service Pack, ShadowExplorer version).

Why are there only Shadow Copies for the c: drive, but not for any other drive?

ShadowExplorer relies on System Protection being enabled on each drive you want to recover anything from. And it must be enabled before any incident you want to revert. Windows enables System Protection by default on the system partition (c:) only. So this time it's probably too late. If you want to be able to recover from incidents on any partition other than c: in the future, you should turn on System Protection for these partitions right away!

Will ShadowExplorer work on Windows 9x/ME/2000/XP?

As far as we know it won't. It's designed to work on Windows Vista/7/8/10 and it seems to work on Windows Server 2003/2008/2008 R2/2012/2012 R2 as well.

What does „Create process failed: code 740. The requested operation requires elevation.” mean?

This was an error in the installer of Version 0.1 if you tried to start ShadowExplorer at the end of the setup. Please update to a more up-to-date version of ShadowExplorer.
